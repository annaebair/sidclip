# SID-CLIP: Synthetic Image Distillation from CLIP

Few-shot image classification via knowledge distillation from a frozen CLIP teacher into a smaller student model, augmented with Kandinsky-generated synthetic training images.

## Setup

```bash
conda env create -f environment.yml
conda activate sidclip
```

## Configuration

**1. Dataset paths** ã update `DATA_ROOTS` and `SYN_ROOTS` in `data.py`:

```python
DATA_ROOTS = {
    'StanfordCars': '/path/to/StanfordCars',
    'Flowers':      '/path/to/flowers102',
    'Food':         '/path/to/food',
    'DTD':          '/path/to/dtd',
}

SYN_ROOTS = {
    'StanfordCars': '/path/to/FakeStanfordCars',
    'Flowers':      '/path/to/FakeFlowers',
    'Food':         '/path/to/FakeFood',
    'DTD':          '/path/to/FakeDTD',
}
```

Synthetic data directories are expected to follow the naming convention `{SYN_ROOT}{shot}shot/train/` (e.g. `FakeStanfordCars8shot/train/`), as produced by `stable_diffusion_gen.py`.

**2. Teacher checkpoint paths** ã `distill.py:_teacher_linear_path()` maps (dataset, teacher) pairs to checkpoint files saved by Step 1 (finetune). The expected filenames are:

```python
('StanfordCars', 'ViT-L/14'): 'checkpoints/stanfordcars_vitl14.pth'
('StanfordCars', 'ViT-B/32'): 'checkpoints/stanfordcars_vitb32.pth'
# ... same pattern for Flowers, Food, DTD
```

These are generated by running `finetune.py --mode last_layer` with `--save_name <name>` matching the expected filename.

**3. TimEfNet checkpoints** ã if using `TimEfNetb0/b1/b2` student models, set the corresponding environment variable pointing to your pretrained checkpoint:

```bash
export TIMEFNETB0_CHECKPOINT=/path/to/timefnetb0.pt
export TIMEFNETB1_CHECKPOINT=/path/to/timefnetb1.pt
export TIMEFNETB2_CHECKPOINT=/path/to/timefnetb2.pt
```

**4. Hugging Face cache** ã if generating synthetic images, set `HF_HOME` in `gen.sh` to a directory where Kandinsky model weights can be downloaded and cached.

## Pipeline

### Step 1: Fine-tune teacher linear head

Train a `LayerNorm + Linear` classification head on top of a frozen CLIP backbone:

```bash
python finetune.py --mode last_layer --model ViT-L/14 --dataset StanfordCars \
  --shot 8 --seed 0 --save_name stanfordcars_vitl14 --lr 1e-4 --epochs 40
```

### Step 2: Generate synthetic images

Uses Kandinsky 2.1 to interpolate between CLIP text and image embeddings:

```bash
python stable_diffusion_gen.py --dataset StanfordCars --shot 8 --syn_shot 300 \
  --start_idx 0 --seed 0
```

### Step 3: Distill

Distill from the CLIP teacher into a student model (EfficientNet or TimEfNet) using KL divergence loss over soft teacher predictions, with synthetic data for the first `syn_epochs` epochs:

```bash
python distill.py --epochs 40 --lr 8e-6 --dist_method kl --batch_size 64 \
  --syn_epochs 30 --model TimEfNetb0 --dataset StanfordCars --shot 8 \
  --syn_shot 300 --seed 0 --teacher ViT-L/14
```

### Baseline (train from scratch, no teacher)

```bash
python train.py --lr 4e-4 --epochs 40 --dataset StanfordCars \
  --model efficientnet_b0 --shot 8 --seed 0
```

## Supported Datasets

| Dataset | Classes |
|---------|---------|
| Stanford Cars | 196 |
| Oxford Flowers102 | 102 |
| Food-101 | 101 |
| DTD | 47 |

## Student Models

- `efficientnet_b0`, `efficientnet_b1`, `efficientnet_b2` ã via `timm`
- `TimEfNetb0`, `TimEfNetb1`, `TimEfNetb2` ã CLIP-trained EfficientNet variants (requires pretrained checkpoints set via environment variables; see Configuration)

## Teacher Models

- `ViT-L/14` ã OpenAI CLIP ViT-Large
- `ViT-B/32` ã OpenAI CLIP ViT-Base

## Key Hyperparameters

| Arg | Default | Description |
|-----|---------|-------------|
| `--T` | 2 | Distillation temperature |
| `--alpha` | 0.9 | Weight on teacher KL loss vs. ground-truth CE |
| `--beta` | 8.0 | DKD non-target class weight |
| `--shot` | ã | Few-shot training size per class (1/2/4/8) |
| `--syn_shot` | ã | Synthetic images per class |
| `--syn_epochs` | 0 | Epochs using synthetic data (real-only after) |
| `--dist_method` | kl | Loss type: `kl` or `dkd` |
